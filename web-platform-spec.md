# WIRE — спецификация Web-платформы и Telegram Mini App

> Единый документ для разработки фронтенда новостной платформы.
> Цель: собрать web + Mini App **одним кодом**, в сильном дизайне, с чистым API-контрактом —
> чтобы потом подключить ядро (база, AI-конвейер, юзерботы) без переделок интерфейса.

---

## 1. Что строим и зачем

Одно приложение, три поверхности:

| Поверхность | Как открывается | Особенности |
|---|---|---|
| **Web** | браузер, `wire.news` | полный кабинет, SEO-страницы событий |
| **PWA** | иконка на телефоне/десктопе | manifest + offline-заглушка |
| **Telegram Mini App** | кнопка в боте | авторизация через `initData`, нативные элементы TG |

**Главный принцип: это ОДИН код.** Никаких «отдельной версии для телеграма».
Разница поверхностей — только адаптер авторизации и обвязка Telegram SDK.

Продуктовая суть: клиент выбирает темы → получает чистую ленту (без рекламы, дубли склеены)
в свою группу/канал и видит ту же ленту на сайте. Мы (операторы) управляем всем тем же интерфейсом.

## 2. Стек

| Слой | Выбор | Почему |
|---|---|---|
| Framework | **React 18 + TypeScript + Vite** | скорость сборки, экосистема, один код на все поверхности |
| Роутинг | **React Router v7** (SPA-режим) | простота; SSR не нужен на старте (SEO-страницы — этап 2) |
| Состояние сервера | **TanStack Query** | кэш, refetch, optimistic updates — вся работа с API через него |
| Локальный стейт | **Zustand** | лёгкий, без бойлерплейта; только UI-состояние (фильтры, модалки) |
| Стили | **CSS-переменные + CSS Modules** | дизайн-токены нативно; без Tailwind — токены и так наши |
| Иконки | **Lucide** | единый стиль, tree-shaking |
| Дата | **Day.js** | лёгкий, локали ru/uk/en |
| Виртуализация | **TanStack Virtual** | лента на тысячи карточек без тормозов |
| Telegram | **@telegram-apps/sdk** | initData, themeParams, MainButton, haptics |
| Тесты | **Vitest + Testing Library** | компоненты и хуки |
| Линт | **ESLint + Prettier** | единый стиль |

Запрещено: тяжёлые UI-киты (MUI, Ant) — ломают дизайн-систему и вес бандла.
Всё из компонентов ниже собирается руками, их немного.

## 3. Дизайн-система «WIRE»

Стиль: **ньюсрум-терминал** — тёмный по умолчанию, плотный, быстрый, с янтарным акцентом.
Светлая тема обязательна и равноценна. Никаких градиентных «AI-фиолетовых» клише.

### 3.1 Токены (CSS-переменные, единственный источник правды)

```css
:root {
  /* фон и поверхности */
  --bg: #F4F5F7;          /* light: подложка */
  --surface: #FFFFFF;      /* карточки */
  --surface-2: #EDEFF2;    /* вложенные панели, thead */
  --ink: #181C23;          /* основной текст */
  --muted: #5C6570;        /* вторичный текст */
  --line: #E1E4E9;         /* разделители */

  /* акцент — янтарь (бренд) */
  --accent: #B26B00;
  --accent-soft: #FFF3DE;
  --accent-line: #E8C988;

  /* семантика (НЕ равна акценту) */
  --good: #2E8F63;  --good-soft: #E3F2EA;
  --warn: #B7791F;  --warn-soft: #FCF0DC;
  --bad:  #C24545;  --bad-soft:  #F9E7E7;
  --info: #2277C4;  --info-soft: #E4F0FB;

  /* телеграм-синий — только внутри TG-контекстов (кнопки «в бота», превью) */
  --tg: #2AABEE;

  /* геометрия */
  --r-card: 14px; --r-control: 9px; --r-chip: 6px;
  --shadow: 0 1px 2px rgba(20,24,32,.06), 0 8px 28px rgba(20,24,32,.07);

  /* типографика */
  --font-ui: "Helvetica Neue", -apple-system, "Segoe UI", Roboto, Arial, sans-serif;
  --font-mono: ui-monospace, "SF Mono", Menlo, Consolas, monospace;
}

[data-theme="dark"] {
  --bg: #0E1219; --surface: #151B24; --surface-2: #1B222D;
  --ink: #E8ECF2; --muted: #94A0AE; --line: #242C38;
  --accent: #F0A83E; --accent-soft: #2A2115; --accent-line: #4A3A1E;
  --good: #4CB98A; --good-soft: #15281F;
  --warn: #E0A83E; --warn-soft: #2A2115;

```

Правила:
- **Ни один компонент не использует hex напрямую** — только токены.
- Тема: `prefers-color-scheme` по умолчанию + ручной переключатель (пишет `data-theme` на `<html>`, хранится в localStorage).
- В Mini App тема наследуется из `themeParams` Telegram: тёмный клиент → тёмная тема.

### 3.2 Типографика

| Роль | Размер/вес | Использование |
|---|---|---|
| H1 | 28–40px / 700, letter-spacing −0.015em | заголовки страниц |
| H2 | 22px / 650 | секции |
| Заголовок карточки | 15–16px / 600 | title новости |
| Body | 14–14.5px / 400, line-height 1.55 | текст |
| Caption | 12–13px / 400, `--muted` | мета, время |
| **Label/цифры** | 10–12px / 600, `--font-mono`, UPPERCASE, letter-spacing .1em | чипы, счётчики, статусы |

Числа в колонках/счётчиках — всегда `font-variant-numeric: tabular-nums`.

### 3.3 Базовые компоненты (UI-kit проекта)

Собрать в `src/ui/` до начала экранов. Каждый — со стори-примером и всеми состояниями.

1. **Button** — primary (акцент), secondary (обводка), ghost, danger; sm/md; loading-спиннер; иконка слева.
2. **Chip** — метка темы. Цветовые варианты по темам (см. 3.4), режим selectable (для онбординга).
3. **Switch** — как в Telegram; disabled-состояние.
4. **Select / Dropdown** — нативный look, поиск внутри для длинных списков (источники).
5. **Input / SearchInput** — с иконкой, clear-кнопкой, ошибкой.
6. **Card** — базовая поверхность (`--surface`, `--r-card`, `--shadow`).
7. **NewsCard** — главный компонент (см. 3.5).
8. **MediaGrid** — 1 фото (16:9), 2 (1×2), 3 (1 большая + 2), 4 (2×2), 5+ (2×2 + бейдж «+N»); видео — превью + ▶ + длительность; lazy-load, blurhash-заглушка.
9. **ImportanceBars** — 5 вертикальных полосок, залито N.
10. **SourceBadge** — «⧉ 14 источников» + разворот списка (кто и во сколько).
11. **Toast** — успех/ошибка, снизу; в Mini App — через haptic + нативный alert где уместно.
12. **Modal / Sheet** — на десктопе модалка, на мобиле шторка снизу (одно API).
13. **Skeleton** — для карточек и списков.
14. **EmptyState** — иллюстрация не нужна: иконка + фраза + действие.
15. **Tabs / SegmentControl** — переключение форматов, периодов.

### 3.5 NewsCard — анатомия главного компонента

```
┌──────────────────────────────────────────────┐
│ [Крипта] [Финансы]            ▌▌▌▌▌ (5/5)    │  ← чипы тем + важность
│ Заголовок события (semibold, 2 строки max)   │

│ [MediaGrid: фото/альбом/видео]               │
│ ⧉ 14 источников · первым @binance · 43 сек   │  ← кликабельно → разворот
│                          [BTC 118 402 ▲1.2%] │  ← контекст-чип (опц.)
│ ┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄ │
```

Состояния: обычная / развёрнутая (полный текст + все источники) / скрытая реклама

Клик по «⧉ N источников» — раскрывает список: канал, время, ссылка на оригинал.

## 4. Экраны и маршруты

### 4.1 Карта маршрутов

```
/                      → лендинг (позже; сейчас redirect на /feed)
/feed                  → Лента (главный экран)
/feed/:eventId         → Событие (карточка + все источники + история)
/routes                → Маршруты доставки (список)
/routes/new            → Визард нового маршрута (3 шага)
/routes/:id            → Настройка маршрута
/sources               → Источники: общий пул (каталог) + мои
/editor/:postId        → Студия «Редактировать»
/settings              → Аккаунт, тема, язык, подписка
/onboarding            → Первый вход (темы → куда → формат)

/ops                   → Пульт оператора (только роль admin)
/ops/pipeline          → Конвейер: статусы, очереди, расход AI
/ops/sources           → Здоровье источников, мёртвые каналы
/ops/quality           → Контроль качества разметки (выборочная проверка)
```

Mini App открывается на `/feed` (или `/onboarding` при первом входе)
через `startapp`-параметр — deep-link на любой маршрут: `?startapp=route_new`.

### 4.2 Лента `/feed`

- **Layout web ≥1024px:** 3 колонки — фильтры (240px) | лента (flex) | контекст (280px).
- **Layout mobile / Mini App:** одна колонка; фильтры — кнопка → шторка; контекст скрыт.
- Слева: чекбоксы тем (со счётчиками за сегодня), фильтры (реклама/дубли/важность/язык),
  «Мои источники», кнопка «+ Бот в мою группу».
- Центр: LIVE-бейдж (WebSocket подключён — зелёный пульс), поиск, лента `NewsCard`
  с виртуализацией. Новые события приезжают сверху с мягкой подсветкой `--accent-soft` 2 сек.
- Справа (web): карточка выбранного источника (аватар, подписчики, «4 108 публикаций»,
  активность) — данные обогащаются TGStat, но UI не зависит от их наличия (graceful).
- Фильтры пишутся в URL (`/feed?topics=crypto,war&imp=3`) — шарибельно, back работает.

### 4.3 Событие `/feed/:eventId`

Полная карточка: весь текст (рендер telegram-entities → HTML), полный альбом,
таблица источников (канал · время · «+3 мин от первого» · ссылка), кнопки:
Редактировать / Отправить в маршрут / Копировать ссылку. Это же — SEO-страница (этап 2).

### 4.4 Маршруты `/routes`

Список карточек-маршрутов (как в макете): назначение (📢 канал / 📣 группа / 💬 ЛС),
чипы тем, формат (оригинал/карточка/дайджест), статус вкл/пауза (Switch),
«сегодня: 212 доставлено». Свайп/меню: пауза, редактировать, удалить (с confirm).

**Визард `/routes/new` — 3 шага, прогресс сверху:**
1. Темы (chips-grid) + фильтры (реклама, дубли, важность, ключевые слова)
2. Куда: выбор из подключённых чатов или «добавьте бота в группу» (инструкция + проверка)
3. Формат: Оригинал / Карточка / Дайджест (радио-карточки с описанием) → «Создать»

В Mini App кнопка «Создать» — это нативная **MainButton** Telegram.

### 4.5 Источники `/sources`

Две вкладки:
- **Каталог** — общий пул: поиск, фильтр по темам, сортировка (подписчики/активность),
  строка: аватар, название, @username, тема-чип, «~40 постов/день», Switch «в моей ленте».
- **Мои** — добавление своего канала: инпут «вставь ссылку или @username» → превью
  (название, подписчики, AI-тема, активность) → «Добавить». Статусы: активен / проверяется /
  отклонён (спам: «78% постов — реклама»). Пустое состояние объясняет ценность.

### 4.6 Студия `/editor/:postId`

Два столбца (mobile: вертикально):
- Слева: оригинал (карточка + медиа) и блок «AI предлагает» — 3 варианта заголовка
  (кликбейт/нейтральный/короткий), клик подставляет.
- Справа: заголовок (input), текст (textarea с тулбаром **Ж К 🔗** + AI-кнопки
  «Сократить / Переписать / Перевести»), медиа (заменить / выбрать из альбома / сгенерировать),
  «Куда отправить» (чипы назначений), кнопки **Опубликовать** / **В отложку**.
- Всё AI — с loading-состоянием на кнопке; результат заменяет текст с возможностью Undo.

### 4.7 Онбординг `/onboarding` (критичен для Mini App)

Три экрана = визард маршрута (4.4), но с текстами для новичка.
Цель: **от «/start» до работающей ленты ≤ 60 секунд, 3 нажатия минимум.**
После финиша — конфетти-хаптик (в TG) и карточка «Готово! Первые новости уже едут».

### 4.8 Пульт оператора `/ops` (роль admin)

Тёмный дашборд (в стиле остального, плотнее):
- **Статусы сервисов**: юзербот(ы), RSS, очередь AI, БД, доставка — зелёный/жёлтый/красный,
  аптайм, последний heartbeat.
- **Цифры дня**: обработано / отсеяно рекламы / склеено дублей / доставлено / расход AI в $.
- **Здоровье источников**: «молчит дольше обычного» (список), слетевшие аккаунты.
- **Качество разметки**: случайные 50 постов → оператор подтверждает/правит метку →
  счётчик точности, исправления уходят в примеры промпта.
Всё read-only через API + пара действий (перезапуск воркера, расклеить событие).

## 5. Telegram Mini App — обязательные требования

1. **Авторизация**: `initData` из `window.Telegram.WebApp` → на бэкенд → проверка подписи →
   JWT-сессия. В web — Telegram Login Widget (тот же аккаунт) или magic-link (этап 2).
2. **Тема**: слушать `themeChanged`; маппить `themeParams` → наши токены (только выбор
   light/dark, палитра остаётся наша — не красить интерфейс в цвета клиента TG).
3. **Viewport**: `expand()` при старте; уважать `safeAreaInset` (iOS: вырез и home-индикатор);
   высота через `viewportStableHeight`, не `100vh`.
4. **MainButton** — главное действие экрана (Создать / Сохранить / Опубликовать);
   **BackButton** — навигация назад по стеку роутера; **HapticFeedback** — на success/error.
5. **ClosingConfirmation** — включать на экранах с несохранённой формой.
6. Не использовать hover-зависимые действия — всё доступно по тапу (панель действий карточки
   открывается по long-press/кнопке «⋯»).
7. Ссылки наружу — `openTelegramLink` / `openLink`, не `<a target="_blank">`.
8. Вес: первый экран ≤ 200KB JS (gzip) — Mini App открывают на слабых сетях.

## 6. API-контракт (фронт строится на нём, бэк подключается позже)

### 6.1 Принципы

- REST JSON, префикс `/api/v1`. Auth: `Authorization: Bearer <jwt>`.
- **Mock-first**: фронт разрабатывается на MSW (Mock Service Worker) с фикстурами
  из этого контракта. Подключение живого API = смена BASE_URL, ноль правок компонентов.
- Пагинация: cursor-based (`?cursor=…&limit=30` → `{ items, next_cursor }`).
- Ошибки: `{ error: { code, message } }`, коды: `unauthorized`, `not_found`,
  `validation`, `rate_limited`, `payment_required`.

### 6.2 Сущности

```ts
type Topic = 'war'|'crypto'|'finance'|'world'|'politics'|'tech'|'sport'|'esport'|'incidents'|'other';

interface Event {              // склеенное событие — то, что видит юзер в ленте
  id: string;
  title: string;               // AI-заголовок или первая строка
  text: string;                // текст лучшего поста (telegram-entities уже в HTML)
  topics: Topic[];
  language: 'ru'|'uk'|'en'|'other';
  importance: 1|2|3|4|5;
  is_ad: boolean;
  media: Media[];
  sources: EventSource[];      // все посты-дубли
  first_source: EventSource;
  created_at: string;          // ISO, время ПЕРВОГО поста
  updated_at: string;
  context?: { kind:'price', label:'BTC', value:'118402', change:+1.2 }; // опц. обогащение
}

interface EventSource {
  channel_id: string; channel_title: string; username?: string;
  post_url?: string; posted_at: string; views?: number; forwards?: number;
}

interface Media {
  kind: 'photo'|'video'|'gif';
  url: string; thumb_url: string; blurhash?: string;
  width: number; height: number; duration_sec?: number; // video
}

interface Route {
  id: string;
  destination: { kind:'channel'|'group'|'dm'; chat_id:string; title:string };
  topics: Topic[];
  filters: { hide_ads:boolean; merge_duplicates:boolean; min_importance:1|2|3|4|5;
             languages:string[]; keywords:string[]; sources_whitelist?:string[] };
  format: 'original'|'card'|'digest';
  digest?: { times:['09:00','21:00'] };
  enabled: boolean;
  stats: { delivered_today:number };
}

interface Source {             // канал в пуле
  id:string; title:string; username?:string; avatar_url?:string;
  topics:Topic[]; posts_per_day:number; subscribers?:number;
  scope:'pool'|'mine';         // общий пул или добавлен клиентом
  status:'active'|'checking'|'rejected'; reject_reason?:string;
  in_my_feed:boolean;
}

interface Me {
  id:string; name:string; tg_username?:string;
  plan:'start'|'pro'|'editorial'|'internal';
  role:'client'|'admin';
  limits:{ routes:number; my_sources:number };
}
```

### 6.3 Эндпоинты

```

GET  /api/v1/destinations      // подключённые чаты (куда бот добавлен)
GET  /api/v1/sources?scope=pool|mine&topic=&q=&cursor=
POST /api/v1/sources/resolve   { url } → превью канала (title, subs, ai_topic, ppd)
POST /api/v1/sources           { url } → добавить в мои
PATCH /api/v1/sources/:id      { in_my_feed }
POST /api/v1/editor/:postId/suggest   { kind:'headlines'|'rewrite'|'shorten'|'translate' }
POST /api/v1/editor/:postId/publish   { title, text, media_ids, destination_ids, schedule_at? }
POST /api/v1/auth/telegram     { init_data } → { jwt }

// пульт (role=admin)
GET  /api/v1/ops/status        // сервисы, heartbeat'ы
GET  /api/v1/ops/metrics/today // счётчики дня + расход AI
GET  /api/v1/ops/sources/health

```

### 6.4 Real-time

- **WebSocket** `/api/v1/ws` (fallback: polling каждые 20с, автопереключение).
- События: `event.new` (полный Event), `event.updated` (доклеился источник → обновить
  бейдж «⧉ N» без перерисовки карточки), `route.delivered` (тикер статистики), `ops.alert`.
- Индикатор LIVE в шапке ленты = состояние сокета.

## 7. Состояния, качество, производительность

### 7.1 У каждого экрана — 4 состояния

`loading` (скелетоны, не спиннер) / `empty` (объясняет и ведёт к действию) /
`error` (что случилось + Retry) / `data`. Пустая лента нового юзера — самое важное
empty-состояние: «Выберите темы — и здесь появятся новости» + кнопка.

### 7.2 Производительность

- Виртуализация ленты (TanStack Virtual), ≥60fps скролл.
- Медиа: lazy + blurhash-плейсхолдеры; `srcset` для ретины; видео — только превью,
  плеер по клику.
- Code-splitting по маршрутам; студия и пульт — отдельные чанки.
- Бюджеты: первый экран ≤200KB gz; LCP ≤2.0s на 4G; интеракция ≤100ms.

### 7.3 Доступность и мелочи качества

- Фокус видим (`:focus-visible`), вся навигация проходима с клавиатуры (web).
- `prefers-reduced-motion` — отключает пульс LIVE, подсветку новых карточек, конфетти.
- Тексты интерфейса — через словарь `src/i18n/` (ru сейчас; uk/en — добавление файла).
- Время: «43 сек назад» до часа, дальше «14:32», вчера+ — дата; тултип с точным временем.
- Все деструктивные действия — confirm (удалить маршрут, отклонить кандидата).

## 8. Структура проекта

```
src/
├─ app/            # роутер, layout'ы (WebShell / MiniAppShell), провайдеры
├─ ui/             # дизайн-система: Button, Chip, NewsCard, MediaGrid, …
├─ features/
│  ├─ feed/        # лента: FiltersPanel, EventList, EventPage, LiveIndicator
│  ├─ routes/      # маршруты: RouteCard, RouteWizard, DestinationPicker
│  ├─ sources/     # каталог, мои источники, resolve-превью
│  ├─ editor/      # студия: OriginalPane, EditorPane, AiSuggest
│  ├─ onboarding/
│  └─ ops/         # пульт оператора
├─ entities/       # типы и API-хуки: useFeed, useRoutes, useSources, useMe
├─ shared/
│  ├─ api/         # клиент, WS, MSW-моки + фикстуры
│  ├─ tg/          # адаптер Telegram SDK (initData, MainButton, theme)
│  └─ lib/         # время, форматирование, telegram-entities → HTML
└─ styles/         # tokens.css, reset, глобальные
```

Правило: `ui/` не знает про API; `features/` не лезут друг в друга напрямую;
всё телеграм-специфичное — только через `shared/tg` (в web этот адаптер = no-op).

## 9. Порядок сборки (без дат — очерёдность)

1. **Фундамент**: токены, темы, UI-kit (все компоненты из 3.3 со состояниями), MSW-моки.
2. **Лента**: NewsCard + MediaGrid + фильтры + виртуализация + WS-индикатор. Самый важный экран.
3. **Маршруты**: список + визард (он же онбординг с другими текстами).
4. **Источники**: каталог + добавление своего (resolve-превью).
5. **Mini App-обвязка**: initData-auth, MainButton/BackButton, тема, viewport, haptics.
6. **Студия редактора**: layout + AI-кнопки (моки) + публикация.
7. **Пульт оператора**: статусы + метрики + качество.
8. **PWA**: manifest, иконки, offline-страница.
9. Подключение живого API вместо MSW, экран за экраном.
