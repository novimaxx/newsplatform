# WIRE — дизайн-бриф для генерации экранов

Продукт: новостной терминал. Собирает 1000+ Telegram-каналов + RSS в единую ленту,
AI размечает по темам/важности, склеивает дубли в события. Три поверхности: Web, Telegram Mini App, Admin.

## Стиль (обязательно из этой системы)
- Направление: `wire-2026.html` — две темы: тёмная «ночной деск» (сине-чёрный + янтарное свечение, стекло),
  светлая «бумага утреннего выпуска» (тёплая бумага, чернила, охра). НЕ инверсия друг друга.
- Акцент один — янтарь (#F5AE3D dark / #A85B00 light). Семантика (good/bad/info) ≠ акцент. TG-синий только в TG-контекстах.
- Плотность высокая: это рабочий терминал, не лендинг. Моноширинные метки, tabular-nums, волосяные линии.
- Токены и компоненты — в foundations/ и components/.

## Экраны для генерации (по приоритету)
1. Лента: master-detail — плотный поток слева, карточка события справа (текст, медиа-сетка, хронология источников «первым сообщил»).
2. Событие: полная карточка со всеми источниками и таймлайном.
3. Маршруты доставки: карточки «тема → чат», статус вкл/пауза, формат.
4. Пульт оператора: KPI со спарклайнами, статусы сервисов, аккаунты-сборщики.
5. Mini App: онбординг 3 шага (темы → куда → формат), мобильный.

## Ключевые компоненты
NewsCard (чипы тем + важность 5 полосок + «⧉ N источников»), медиа-сетка (альбом «+N», видео),
переключатель лент (Моя лента / Общий поток / Готовые посты), стеклянный топбар с live-тикером.
